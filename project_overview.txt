# AI-NutriCare - Complete Project Manual & Overview

## 1. Project Introduction
**AI-NutriCare** is a full-stack, AI-powered medical intelligence application. It allows users to upload their medical lab reports (blood work, diagnostics) and uses advanced Artificial Intelligence to extract, structure, and analyze the biomarkers. It then synthesizes this clinical data alongside the user's physical metrics (Age, Weight, Height) to generate a personalized "Nutritional Protocol" (diet plan) and highlights any critical health abnormalities. 

It also features a built-in RAG (Retrieval-Augmented Generation) Chatbot that serves as a personalized clinical assistant, allowing users to ask natural language questions about their specific medical data.

---

## 2. Core Features (What it does)
1. **Secure Authentication**: Users can sign up and log in securely. Their information and previous clinical data are tied to their account.
2. **Medical Document Upload & OCR**: Users upload lab reports (PDF, JPG, PNG). The system reads the text from these files using OCR (Optical Character Recognition).
3. **AI Biomarker Extraction Engine**: An LLM (Large Language Model - Google Gemini) reads the messy OCR text and structures it into a clean JSON format, extracting everything from Patient Name to specific biological tests (e.g., Hemoglobin, Cholesterol) and their expected reference ranges.
4. **Abnormal Findings Detection**: The system cross-references the patient's results with medical reference ranges. It has a robust fallback engine that scans every extracted test to explicitly flag anything interpreted as "high", "low", or "critical", displaying this prominently.
5. **AI Nutritional Synthesis**: By calculating the patient's BMI and analyzing their blood work, the AI generates a customized diet plan including:
   - Executive Summary of their health status
   - Focus Superfoods to improve deficient markers
   - Foods to Minimize/Avoid
   - A Daily Precision Meal Blueprint
6. **Dynamic PDF Generation**: Users can download a beautifully structured PDF summarizing their patient profile, critical abnormalities, and their custom diet plan.
7. **Interactive RAG Chatbot**: A floating chat widget is available on the dashboard. Once a report is processed, the chatbot "reads" it and can answer any specific questions the user has about their health data.

## 3. Technology Stack (How it's built)

### Frontend (Client-Side)
- **Framework**: React.js (Bootstrapped via Vite for fast performance)
- **Routing**: React Router DOM (Handles navigation between Login, Signup, Dashboard)
- **Styling**: Pure CSS (`App.css`) using a modern "Claymorphism" design aesthetic (soft pastel colors, puffy drop-shadows, glass-like input fields). It is fully responsive across desktop, tablet, and mobile.
- **State Management**: React Context API (`AuthContext` for user sessions) and local state hooks (`useState`, `useEffect`).

### Backend (Server-Side)
- **Framework**: FastAPI (Python) - chosen for its extreme speed and built-in asynchronous capabilities.
- **Server**: Uvicorn (ASGI web server implementation for Python).
- **Database**: PostgreSQL (Relational database for storing user accounts and hashed passwords).
- **ORM**: SQLAlchemy (Maps Python classes to Database tables).
- **Authentication**: JWT (JSON Web Tokens) for secure, stateless user API requests, combined with bcrypt for password hashing.
- **PDF Generation**: ReportLab (A robust Python library used to programmatically draw and generate the downloadable PDF reports).

### AI & Machine Learning Tools
- **LLM/Extraction**: Google Gemini API is heavily utilized to parse raw text into structured JSON data (`llm_extractor.py`).
- **Vector Database**: ChromaDB (Used to create semantic embeddings of the medical reports so the Chatbot can quickly search and retrieve relevant facts to answer user questions).
- **Embeddings**: Gemini Embeddings (Used to power the Retrieval-Augmented Generation RAG flow).

## 4. Key Logic Flows (Explaining to teammates)

### "How does the document upload work?"
When a user uploads a file and enters their metrics (Age, Weight, Height), the React frontend sends a `FormData` request to the FastAPI backend (`analyze-report` endpoint). The backend extracts the text, sends the text to the Gemini API with a strict prompt to map tests and interpretations, calculates the BMI, asks Gemini to generate a diet plan based on those results, and sends all of this structured data back to the React UI to be displayed in the Bento-box layout.

### "Why don't we ask for the lab name anymore?"
In the final version, we removed explicit UI inputs for the Lab Name and Report Status. Instead, we rely completely on the AI extraction engine to intelligently find the laboratory name (e.g., "Quest Diagnostics") and the document status directly from the scanned text.

### "How does the Chatbot know about my health?"
This uses a RAG (Retrieval-Augmented Generation) pipeline. When the report is analyzed, the structured data and diet plan are temporarily stored in the Chatbot's context state on the frontend. When the user asks a question, the entire structured health profile is appended to the system prompt, allowing the AI to answer specifically using the provided clinical context wrapper.

### "What happens if the AI misses critical abnormal results?"
We built a defensive fallback in `api.py`. If the AI fails to explicitly create the `abnormal_findings` list, our Python code iterates over every single extracted test. If it sees any interpretation string containing "low", "high", "abnormal", or "critical", it forcefully extracts it and injects it into the abnormal findings array to ensure the user is always warned about red flags.

## 5. File & Directory Breakdown (What each file does)

### Frontend Directory (`frontend/src/`)
* **`main.jsx` & `App.jsx`**: The foundational entry points of the React application. They handle the main routing rules (directing users to `/login`, `/signup`, or `/dashboard`).
* **`App.css`**: The global stylesheet. Contains all of the modern "Claymorphism" aesthetic variables, animations, and the responsive design `@media` queries that make the app look great on phones and tablets.
* **`pages/Dashboard.jsx`**: The core User Interface where users upload their documents, view their parsed "bento box" medical data, and open the RAG Chatbot. 
* **`pages/Login.jsx` / `pages/Signup.jsx`**: The authentication screens where users create accounts and sign in.
* **`context/AuthContext.jsx`**: A global state manager that keeps track of whether a user is securely logged in across all pages using their JWT token.
* **`api/reportApi.js`**: A utility network file. It manages the `fetch` / `axios` calls that send data from the React frontend to the FastAPI backend (such as sending the uploaded PDF or requesting chatbot replies).

### Backend Directory (`backend/`)
* **`api.py` (or `main.py`)**: The central entry point for the FastAPI server. It defines all the endpoints like `/analyze-report/` (for document uploads), `/chat/` (for the RAG bot), and handles the incoming requests.
* **`database.py`**: Sets up the connection to the PostgreSQL database.
* **`models_auth.py`**: Defines the SQLAlchemy database structure (schemas) for the User tables.
* **`schemas.py`**: Contains Pydantic models. These strict validation rules ensure that any data coming in or out of the API is correctly formatted.
* **`auth.py`**: Contains the security logic for the application, including hashing user passwords with bcrypt and generating JWT login tokens.
* **`prompts.py`**: Contains the specific, heavily-engineered text instructions sent to the Google Gemini AI. This is where we tell the AI exactly *how* to extract the medical data and output it as JSON.

### Backend Modules (`backend/modules/`)
* **`llm_extractor.py`**: Connects directly to the Google Gemini API to parse the raw OCR text of the lab report into the clean, structured patient profile.
* **`bmi_calculator.py`**: A helper function that takes the user's Age, Weight, and Height to categorize their physical baseline.
* **`diet_generator.py`**: Connects to the AI to synthesize the patient's blood work alongside their BMI category to create the customized "Nutritional Protocol" (e.g., Focus Superfoods).
* **`pdf_exporter.py`**: Uses the `ReportLab` library to programmatically draw and arrange the extracted medical data and custom diet plan into the beautiful, downloadable PDF.
* **`rag_chatbot.py`**: Translates the user's natural language questions into the prompt context wrapper so the AI can answer specific questions based on the uploaded report.

## 6. Dependencies & Installation Guide
If a new developer or tester wants to run this project on their own machine, they must install the following libraries and tools.

### Prerequisites (System Requirements)
1. **Node.js** (v18+ recommended) - Required to run the React frontend.
2. **Python** (v3.10+ recommended) - Required for the FastAPI backend.
3. **PostgreSQL** - Must be installed locally (or via Docker) to store user authentication data.

### Frontend Dependencies (React)
The frontend relies on the Node Package Manager (NPM). The core libraries include:
- `react`, `react-dom` (UI Library)
- `react-router-dom` (Routing and Navigation)
- `axios` (For making HTTP requests to the FastAPI backend)
- `vite` (The build tool and development server)
- `tailwindcss` (Utility framework, though custom CSS `App.css` handles most of the Claymorphism UI)

**Frontend Setup Steps:**
1. Open a terminal and navigate to the `frontend/` directory.
2. Run `npm install` to download all the libraries listed in `package.json`.
3. Run `npm run dev` to start the frontend server (runs on `http://localhost:5173`).

### Backend Dependencies (Python FastAPI)
The backend requires a Python Virtual Environment (`venv`) to keep dependencies isolated. The heavy lifting libraries are:
- **Core Server**: `fastapi`, `uvicorn` (ASGI server), `python-multipart` (for file uploads)
- **Database & Auth**: `SQLAlchemy` (ORM), `psycopg2-binary` (Postgres driver), `passlib[bcrypt]` (password hashing), `python-jose[cryptography]` (JWT tokens)
- **AI & ML Tools**: `google-generativeai` (Gemini LLM API), `chromadb` (Vector semantic database for the Chatbot)
- **PDF Generation**: `reportlab`

**Backend Setup Steps:**
1. Open a terminal and navigate to the `backend/` directory.
2. Create a virtual environment: `python -m venv venv`
3. Activate the virtual environment:
   - *Windows*: `venv\Scripts\activate`
   - *Mac/Linux*: `source venv/bin/activate`
4. Install all dependencies using the requirements file: 
   `pip install -r requirements.txt`
   *(Alternatively, if installing manually: `pip install fastapi uvicorn sqlalchemy psycopg2-binary reportlab passlib[bcrypt] python-jose[cryptography] python-multipart google-generativeai chromadb`)*
5. Configure Environment Variables: Create a `.env` file in the `backend/` directory referencing your local PostgreSQL database and your Gemini API Key:
   ```env
   DATABASE_URL=postgresql://username:password@localhost/dbname
   SECRET_KEY=your_secret_crypto_string
   GEMINI_API_KEY=your_google_ai_studio_key
   ```
6. Start the backend: `uvicorn main:app --reload --port 8001` (Ensure it runs on 8001, as the frontend expects this port).

If both the frontend and backend are running, and the `.env` variables are correct, the full application will function smoothly on your local machine!
